| 字段 | 类型 | 说明 | 示例值 |
|------|------|------|--------|
| `season` | List<String> | 适用季节 | ["春", "夏", "秋", "冬"] |
| `budgetLevel` | String | 预算等级 | free/budget/moderate/premium/luxury |
| `duration` | String | 建议时长 | 1小时/半天/全天/2天 |
| `bestTime` | String | 最佳时间 | 早晨/下午/傍晚/夜晚 |
| `crowdLevel` | String | 拥挤度 | 低/中/高 |

| `nearbyPOIs` | List<String> | 周边兴趣点 | ["西湖断桥", "雷峰塔"] |

#### 自动推断实现

实现了 8 个智能推断方法：

1. **`inferSeason()`** - 根据内容推断适用季节
   - 关键词匹配：樱花→春，海滩→夏，红叶→秋，滑雪→冬
   - 默认值：全年适宜

2. **`inferBudgetLevel()`** - 推断预算等级
   - 检测价格相关关键词
   - 5 级分类：free/budget/moderate/premium/luxury

3. **`inferDuration()`** - 推断建议时长
   - 匹配时间表达
   - 按主题类型默认值

4. **`inferBestTime()`** - 推断最佳游览时间
   - 识别时间偏好关键词
   - 主题相关默认值

5. **`inferCrowdLevel()`** - 推断拥挤度
   - 热门/小众关键词检测
   - 基于主题的默认值

6. **`extractLocation()`** - 提取具体位置
   - 地址模式识别
   - 正则表达式匹配

7. **`extractPriceRange()`** - 提取价格范围
   - 支持多种价格格式：¥100-200, ￥50-100元
   - 正则表达式提取

8. **`extractFacilities()`** - 提取设施标签
   - 8 种常见设施识别
   - WiFi/停车场/早餐/泳池/健身房/SPA/餐厅/电梯/无障碍

### ✅ 任务 2: 混合检索基础框架

**实施时间**: 2026-04-11  
**优先级**: P0  
**状态**: 基础框架已完成

#### 实现方案

**混合检索策略**：
```
最终分数 = α × BM25分数 + β × 向量分数
```

**默认权重**：
- BM25 权重 (α): 0.4
- 向量权重 (β): 0.6

#### 核心组件

**1. BM25 关键词检索**

```java
// BM25 参数
K1 = 1.5  // 词频饱和参数
B = 0.75  // 文档长度归一化参数

// BM25 公式
Score(q,d) = Σ IDF(qi) × (TF(qi,d) × (K1 + 1)) / (TF(qi,d) + K1 × (1 - B + B × |d|/avgdl))
```

**特性**：
- ✅ 完整实现 BM25 算法
- ✅ 支持中英文分词
- ✅ 文档长度归一化
- ✅ 词频饱和处理
- ✅ 逆文档频率计算

**2. 向量检索**

- 使用 Milvus 向量数据库
- 语义相似度匹配
- 支持元数据过滤

**3. 加权融合**

```java
// 归一化
normalized_score = (score - min) / (max - min)

// 融合
hybrid_score = 0.4 × bm25_normalized + 0.6 × vector_normalized
```

**特性**：
- ✅ Min-Max 归一化
- ✅ 可配置权重
- ✅ 自动排序

#### 代码实现

**新文件**：
- `HybridRetrievalService.java` (294 行)
  - `hybridSearch()` - 主检索方法
  - `computeBM25Scores()` - BM25 分数计算
  - `computeVectorScores()` - 向量分数计算
  - `hybridRank()` - 混合排序
  - `tokenize()` - 中英文分词

**修改文件**：
- `TravelKnowledgeVectorStoreRepository.java`
  - 添加 `toSnippetStatic()` 静态方法

#### 使用示例

```java
@Autowired
private HybridRetrievalService hybridRetrievalService;

// 执行混合检索
TravelKnowledgeRetrievalResult result = hybridRetrievalService.hybridSearch(
    "杭州",                    // 目的地
    List.of("relaxed", "food"), // 偏好
    "西湖附近的美食推荐",        // 查询
    10                         // 返回数量
);
```

- 精确匹配地名能力提升

## 🔧 配置选项

### 混合检索权重

```yaml
# application.yml
travel:
  agent:
    retrieval:
      hybrid:
        bm25-weight: 0.4    # BM25 权重
        vector-weight: 0.6  # 向量权重
        bm25:
          k1: 1.5           # 词频饱和参数
```
