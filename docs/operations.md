# Operations Guide

This document centralizes the runtime, script, log, and export conventions for the repository.

## Script Index

- `scripts/preflight-travel-agent.ps1`
  - validates local prerequisites and configuration before startup
- `scripts/start-travel-agent.ps1`
  - starts the backend and, optionally, the frontend preview and MCP server
- `scripts/stop-travel-agent.ps1`
  - stops processes launched by the start script by reading PID files
- `scripts/release-smoke-travel-agent.ps1`
  - runs a release-style backend smoke check and optional frontend build verification
- `scripts/export-feedback-dataset.ps1`
  - exports raw feedback records to `data/exports/feedback-dataset.json` by default
- `scripts/analyze-feedback-loop.ps1`
  - exports aggregated feedback analysis to `data/exports/feedback-loop-summary.json` and `data/exports/feedback-loop-report.md`

## Runtime Layout

- `logs/runtime/`
  - default stdout, stderr, and PID files for local start / stop flows
- `logs/release-smoke/`
  - stdout and stderr generated by `release-smoke-travel-agent.ps1`
- `logs/archive/`
  - historical logs moved out of the repository root or legacy locations
- `data/exports/`
  - manually exported feedback datasets and feedback analysis reports
- `data/travel-agent.db`
  - local SQLite database used by the application
- `data/milvus/`
  - local Milvus state when vector retrieval is enabled

## Current Conventions

- Runtime logs should go under `logs/`, not the repository root.
- Feedback exports should go under `data/exports/`, not mixed into runtime logs.
- Generated files under `logs/` and `data/exports/` are git-ignored.
- If you introduce a new script that writes files locally, prefer one of these directories instead of inventing a new root-level drop location.

## Common Commands

Preflight:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\preflight-travel-agent.ps1
```

Local startup:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start-travel-agent.ps1 -Build -StartFrontend -RunPreflight -ToolProvider LOCAL
```

Shutdown:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\stop-travel-agent.ps1
```

Release smoke:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\release-smoke-travel-agent.ps1
```

Feedback export:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\export-feedback-dataset.ps1
```

Feedback analysis:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\analyze-feedback-loop.ps1
```

## Cleanup Notes

- Before publishing, make sure local runtime logs in `logs/` are not mixed into a commit.
- If you need to inspect old logs after cleanup, check `logs/archive/`.
- If a script still writes to an unexpected root-level path, treat that as a repository hygiene issue and move it under `logs/` or `data/exports/`.
