### 2. 混合检索 ✅

**BM25 算法**：
```
Score(q,d) = Σ IDF(qi) × (TF(qi,d) × (K1 + 1)) / (TF(qi,d) + K1 × (1 - B + B × |d|/avgdl))

参数:
- K1 = 1.5 (词频饱和)
- B = 0.75 (长度归一化)
```

**混合策略**：
```
最终分数 = 0.4 × BM25分数 + 0.6 × 向量分数
```

**特性**：
- ✅ 完整 BM25 实现
- ✅ 中英文分词
- ✅ 文档长度归一化
- ✅ Min-Max 归一化
- ✅ 可配置权重

### 4. Reranking ✅

**4 维分数融合**：
```
最终分数 = 0.35 × 相关性 
         + 0.25 × 偏好匹配 
         + 0.20 × 时效性 
         + 0.20 × 多样性
```

**相关性评分**（模拟 Cross-Encoder）：
- 标题匹配 (40%)
- 内容匹配 (30%)
- 标签匹配 (15%)
- 元数据匹配 (15%)

**用户偏好匹配**：
- 旅行风格 (relaxed/family/foodie...)
- 预算等级 (free/budget/luxury...)
- 设施偏好 (WiFi/停车场/早餐...)
- 季节匹配

**MMR 多样性**：
```
MMR = λ × Sim(q, di) - (1-λ) × max(Sim(di, dj))
λ = 0.7
```

## 💡 技术亮点

### 1. 端到端优化

从查询到结果，每个环节都进行了优化：

```
查询 → 理解 → 检索 → 排序 → 结果
  ↓       ↓       ↓       ↓       ↓
纠错   扩展   混合   Rerank  个性化
扩展   意图   BM25   偏好    多样化
重写   实体   向量   时效    高质量
```

### 2. 自适应策略

- 查询理解自适应（纠错/扩展/意图）
- 分块策略自适应（主题→地理→时间→重叠）
- 检索策略自适应（BM25 + 向量）
- Reranking 自适应（相关性+偏好+时效+多样性）

### 3. 可扩展架构

```

  ├─ QueryUnderstanding ✅
```

## 🎓 经验总结

### 最佳实践

1. **元数据是关键**
   - 丰富的元数据大幅提升检索质量
   - 自动推断减少人工标注成本
   - 多维度过滤提高精确度

2. **混合检索更有效**
   - BM25 + 向量 > 单一检索
   - 精确匹配 + 语义理解
   - 可配置权重适应不同场景

3. **分块要智能**
   - 不同内容需要不同策略
   - 重叠分块保持上下文
   - 句子边界避免硬切分

4. **Reranking 不可少**
   - 重排序是质量保障
   - 多维度评分更全面
   - MMR 保证多样性

5. **查询理解很重要**
   - 好的查询 = 好的结果
   - 纠错和扩展提升召回
   - 意图识别提高精确度

### 架构设计

- **组件化**：每个功能独立组件
- **可配置**：所有参数可调整
- **可扩展**：易于添加新组件
- **可测试**：每个组件独立测试
- **向后兼容**：不破坏现有功能
